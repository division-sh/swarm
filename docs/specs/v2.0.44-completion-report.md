## 2) Anti-Bias Assignment Implemented
Updated: `internal/runtime/pipeline_coordinator.go`

Behavior now:
- On `scoring.requested` emission from derived context (`mode=derived` or derivation lineage markers), runtime extracts generator from discovery context (`generator_agent_id`).
- If generator exists, runtime tries to select an alternate subscribed analysis recipient for `scoring.requested`.
- If alternate exists:
  - emits `scoring.requested` via `PublishDirect(...)` to that single alternate recipient.
  - payload now includes:
    - `assigned_analysis_agent_id`
    - `excluded_analysis_agent_id`
- If no alternate exists:
  - runtime logs explicit anti-bias fallback warning
  - falls back to normal publish (non-blocking behavior)

Schema updates:
- Updated `internal/runtime/event_emit_tools.go` for `scoring.requested`:
  - `mode` enum includes `corpus` and `derived`
  - optional fields:
    - `assigned_analysis_agent_id`
    - `excluded_analysis_agent_id`
